# Git LFS pointer files

This repository does not commit real video model weights. It now includes `.gitattributes` rules so that if a maintainer intentionally tracks large binary assets with Git LFS, Git stores pointer files in the repository instead of normal Git blobs.

## What a valid LFS pointer looks like

A valid Git LFS pointer file is generated by Git LFS. It usually looks like this:

```text
version https://git-lfs.github.com/spec/v1
oid sha256:<sha256-of-real-object>
size <bytes>
```

Do not hand-write fake pointer files. A fake pointer will not contain a real object in LFS storage, so `git lfs pull` cannot restore the model.

## Check pointer coverage

After installing Git LFS locally:

```bash
git lfs install
git lfs track
cat .gitattributes
```

To check whether large files are tracked as LFS pointers:

```bash
git lfs ls-files
```

To verify that pointer files can be resolved into actual local objects:

```bash
git lfs fsck
```

## Recommended model workflow

Use the downloader instead of committing weights:

```bash
make models-list
make models-wan
cat .env.models.local
```

Then copy the generated model path into `.env`, for example:

```bash
VIDEO_BACKEND=wan
WAN_MODEL_PATH=/absolute/path/to/aivideo/models/wan/Wan2.1-T2V-1.3B-Diffusers
```

## When to use Git LFS in this project

Use Git LFS only for small shared reference assets, tiny demo clips, or private deployment repositories where storage and license terms are clear. Do not use this public repository to redistribute gated or license-restricted model weights.
